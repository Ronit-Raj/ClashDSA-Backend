Coin Change

You are given an array of n coin denominations and a target integer amount. Find the minimum number of coins needed to make up exactly amount. You may use each coin denomination an unlimited number of times.

If it is impossible to make up the target amount using the given coins, return -1.

Input Format

  • The first line contains an integer n — the number of coin denominations.
  • The second line contains n space-separated integers representing the coin denominations.
  • The third line contains a single integer amount — the target amount.

Output Format

Print a single integer: the minimum number of coins required to make up amount, or -1 if it is not possible.

Constraints

  • 1 <= n <= 12
  • 1 <= coins[i] <= 2^31 - 1
  • 0 <= amount <= 10^4
  • Time limit: 2 seconds

Example 1

Input

3
1 2 5
11

Output

3

Example 2

Input

1
2
3

Output

-1

Example 3

Input

1
1
0

Output

0

Explanation

In Example 1, the amount 11 can be made up as 5 + 5 + 1, using 3 coins. No combination uses fewer coins.

In Example 2, no combination of the single coin denomination 2 can produce the odd target 3, so the answer is -1.

In Example 3, the target is 0, which requires zero coins regardless of the available denominations.

Notes

  • A standard bottom-up dynamic programming solution runs in O(n * amount) time and O(amount) space.
  • Because coin values can be very large, coins greater than amount can be safely ignored.
