Trapping Rain Water

You are given n non-negative integers representing an elevation map where the width of each bar is 1. Compute how much water it can trap after raining.

Formally, for each index i, the amount of water it can hold is determined by the minimum of the tallest bar to its left and the tallest bar to its right, minus its own height. Water can only be trapped between bars, not on the outermost edges.

Input Format

  • The first line contains a single integer n — the number of bars in the elevation map.
  • The second line contains n space-separated non-negative integers height[0], height[1], ..., height[n-1], where height[i] is the height of the i-th bar.

Output Format

Print a single integer — the total units of water that can be trapped.

Constraints

  • 1 <= n <= 2 * 10^4
  • 0 <= height[i] <= 10^5
  • Time limit: 2 seconds

Example 1

Input

12
0 1 0 2 1 0 1 3 2 1 2 1

Output

6

Example 2

Input

6
4 2 0 3 2 5

Output

9

Example 3

Input

5
3 0 2 0 4

Output

7

Notes

  • For Example 1, the trapped water units are located at indices 2, 4, 5, 6, 9 — totalling 6 units.
  • An O(n) solution exists using the two-pointer technique or a monotonic stack. An O(n) solution using prefix and suffix maximum arrays is also acceptable.
  • A bar of height 0 does not block water; it simply contributes nothing to the left or right maximum.
